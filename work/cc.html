Week 1: Button
In week 1, I'm trying to control the state of a servo using button. I used Fritzing to draw the circuit diagram. The overall circuit utilizes an input and output pin from the Arduino board. The input pin reads digital signal of the button state. When the button is pressed/High, the servo rotates clockwise continuously. While the button is released/Low, the servo hand rotates back to the initial position. In addition, I use a 330Î© resister to protect the circuit and connect it between the button and ground.

Week 2-3: Sensors and Serial Communication
In this project, I tried to control the state of two LEDs on the breadboard via serial communication between Arduino and Processing. The master of communication is the Processing snippet we wrote on the laptop: according to the position of mouse, if the mouse hovers to the red rectangular area, the red LED on the breadboard lights up. When the mouse hovers onto the green rectangle display on Processing, the green LED lights up instead. If the mouse is outside of either rectangular areas. Week 4-6: Arduino Midterm
In the two-week period, I designed a simple shooting game using Arduino and Processing. The character is controlled by a joystick connected to the Arduino board. To make the game more interesting, I designed a penguin world from scratch and all the game elements. Players can control the penguin to move, shoot and trespass various areas such as water, rock, wall, and grass. When two computers can connected, players can setup a server with multiple clients to play the game together.

To implement the game, we used the serial communication between Arduino and Processing with the same technique as the previous week. The joystick sends a single command through serial channel to the Processing terminal. When received the encoded commands, the Processing application extract the moves and run the game on screen. Here we used serial communication because the game is played sequentially so that serial communication can best ensure the command to be delivered in the correct order. Week 8-10: openFrameworks
For the span of 3 weeks, we covered the basics of openFrameworks and fundamentals of Object-Oriented Programming (OOP). For the focus of the class period and homework, we examine into the particle system and kept improving on this area for the following weeks

This simple line animation consists of a particle system that follows the position and direction of the mouse. The particle system consists of 1000 randomly positioned circular objects that constantly draw on the screen. Depending on the speed of each object, you will be able to see the line kind of gain the fluid silky texture.

Final: openFrameworks
For the final project of openFrameworks, I used the physical library Box2d for openFrameworks in order to make the world more realistic. The game is called Space Junk, in the hope of raising the attention to protecting the environment. In this game, the player has to draw lines using their mouse on the white canvas in order to direct the circles to go into the trash can. The whole path cannot go across other spatial objects in the window. The lines created by mouse are converted to edges in the Box2D world so that the dynamic system in Box2d can take care of all the physics in the world.
